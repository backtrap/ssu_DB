 * 오라클 시스템의 DBA 계정
 1. system
 2. sys

 system
거의 모든 DBA 역할 수행
시스템의 booting 및 shutdown 관련 작업 제외

 sys
DBA 자격을 가지는 계정 (SYSDBA privilege)
데이터 사전을 구성하는 테이블 및 뷰를 소유하고 있음

테이블 및 뷰는 시스템을 운영하는데 아주 중요한 역할을 하므로
사용자에 의하여 변경되지 않아야 한다 (sys 포함)

일반적인 환경에서는 특별한 이유가 없으면
sys 계정을 사용하지 않는다

대부분의 DBA 역할을 system 계정을 통하여 수행한다


 * 오라클 프로세스
오라클 시스템은 내부적으로
다수 개의 프로세스를 이용하여 데이터베이스 서버 작업을 수행

 서버 프로세스
사용자 클라이언트 프로세스를 담당하여
클라이언트 요구사항을 처리

 dedicated server process
사용자 프로세스 하나만을 서비스

 shared server process
다수 사용자 프로세스를 서비스


 * 연결과 세션
클라이언트 프로그램이 데이터베이스 서버에 접근하기 위해서는
데이터베이스 서버에 연결해야 한다

 지역 연결 (local)
ID와 암호만 명시

 네트워크를 통한 원격 서버 연결
사용자 계정 및 암호 외에도
호스트 이름도 명시
포트 번호도 명기해야 할 때도 있다
포트 번호의 default 값은 1521

원격 호스트에는
Oracle Net Listener가 특정 포트를 listen하고 있으며
정당한 연결 요청을 받으면
이를 오라클 서버에 전달
그 이후에는 클라이언트와 서버가 직접 통신


 * 외래키

외래키 제약의 이름을 명시하지 않으면
시스템이 자동적으로 이름을 부여한다
이 때 사용자는 데이터 사전을 검색해서 확인할 수 있다

테이블 간에 외래키가 선언되어 있으면
데이터 로딩이 자유롭지 않을 수 있다

데이터 입력 후에 외래키를 선언하는 경우
입력된 값이 제약에 만족하지 않으면 외래키 선언을 할 수 없다


 * 데이터 사전
메타 데이터로서
데티어베이스에 대한 다양한 정보를 기록 관리

데이터베이스에 대한 데이터

사용자에게 주로 테이블 or 뷰 형식으로 정보를 공개

데이터 사전은 DBA가 소유

일반 사용자는 읽기 연산만 수행 가능
데이터 변경 연산 (update, delete, update statements)은 권한이 없다

 Dynamic performance table
데이터베이스 시스템의 활동 현황을 기록하는 가상테이블 (뷰)

데이터 사전을 구성하는 테이블 or 뷰의 명칭
 1. 접두사 user
 2. 접두사 all
 3. 접두사 dba

ex) user_tables


 * 시퀀스  Sequence
여러 사용자 간에서 유일한 값을 생성하여 사용하고자 할 때 유용

시퀀스는 트랜잭션의 성공여부와 상관없이 생성

시스템은 시퀀스 값의 유일성은 보장하나
값의 연속성은 보장 x

시퀀스는 특정 테이블에 연관되어 생성되지 x

시퀀스는 생성문과 동시에 시퀀스 값이 생성되지 x
nextval을 호출해야 시퀀스 값이 생성된다
